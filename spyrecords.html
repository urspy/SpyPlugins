Overview

SpyRecords is a comprehensive Minecraft plugin for SpyMC that tracks player kill/death statistics across multiple servers with different game modes, kits, or arenas. All statistics are combined into one universal record per player, providing a complete cross-server performance analysis system.

Key Features

  • Cross-Server Synchronization: Universal player records shared across your entire server network
  • Global Statistics: Combined kill/death tracking from ALL servers, game modes, kits, and arenas
  • Global Leaderboards: Rankings that span your entire server network
  • PlaceholderAPI Integration: Display statistics anywhere with 8+ available placeholders
  • Real-Time Updates: Instant synchronization across all connected servers
  • MariaDB Backend: Reliable, scalable database solution for large networks
  • Performance Optimized: Asynchronous operations with minimal server impact

Quick Start

Installation

  1. Download the latest SpyRecords.jar (5.3MB) from releases
  2. Place the JAR file in your server's plugins/ folder
  3. Start your server to generate the configuration files
  4. Configure your database settings in config.yml
  5. Restart your server to activate cross-server functionality

Basic Configuration

# config.yml
storage:
  type: 'mariadb'  # Use 'json' for single server
  mariadb:
    host: 'your-database-host.com'
    database: 'spyrecords_global'
    username: 'spyrecords_user'
    password: 'your_secure_password'

PlaceholderAPI Integration

Basic Statistics

  • %spyrecords_kills% - Player's total kills (cross-server)
  • %spyrecords_deaths% - Player's total deaths (cross-server)
  • %spyrecords_kdr% - Player's kill/death ratio
  • %spyrecords_ratio% - Alternative KDR placeholder
  • %spyrecords_name% - Player's recorded name

Ranking Placeholders

  • %spyrecords_rank_kills% - Global kills leaderboard position
  • %spyrecords_rank_deaths% - Global deaths leaderboard position
  • %spyrecords_rank_kdr% - Global KDR leaderboard position

Cross-Server Setup

How It Works

  1. Shared Database: All servers connect to one MariaDB database
  2. Universal Records: Each player has ONE record across all servers
  3. Real-Time Sync: Statistics update instantly across your network
  4. Combined Display: Shows total stats from ALL servers and game modes

Setup Instructions

1. Database Setup

CREATE DATABASE spyrecords_global;
CREATE USER 'spyrecords_user'@'%' IDENTIFIED BY 'your_secure_password';
GRANT ALL PRIVILEGES ON spyrecords_global.* TO 'spyrecords_user'@'%';
FLUSH PRIVILEGES;

2. Configuration (All Servers)

storage:
  type: 'mariadb'
  mariadb:
    host: 'your-database-host.com'
    port: 3306
    database: 'spyrecords_global'
    username: 'spyrecords_user'
    password: 'your_secure_password'
    ssl: false

3. Network Architecture

Technical Specifications

System Requirements

  • Minecraft Version: 1.16.5+
  • Java Version: 8+
  • Server Software: Spigot, Paper, or compatible
  • Database: MariaDB 10.3+ (for cross-server) or Local JSON storage
  • RAM: Minimum 512MB allocated to Minecraft server

Performance Metrics

  • JAR Size: 5.3MB (includes all dependencies)
  • Database Operations: Fully asynchronous
  • Memory Usage: <50MB additional RAM usage
  • Network Load: Minimal - only essential data synchronization

Dependencies

  • Required: None (all dependencies included)
  • Optional: PlaceholderAPI (for placeholder functionality)
  • Optional: HolographicDisplays (for hologram leaderboards)

Configuration Reference

Complete Configuration File

# SpyRecords Configuration
storage:
  type: 'mariadb'  # Options: 'json', 'mariadb'
  
  # JSON Storage (Single Server)
  json:
    file_name: 'player_records.json'
  
  # MariaDB Storage (Cross-Server)
  mariadb:
    host: 'localhost'
    port: 3306
    database: 'spyrecords'
    username: 'spyrecords'
    password: 'password'
    ssl: false
    connection_timeout: 10000
    socket_timeout: 10000

# PlaceholderAPI Settings
placeholderapi:
  enabled: true
  decimal_places: 2
  no_data_value: "0"

# Message Customization
messages:
  prefix: '&8[&eSpyRecords&8] '
  stats_header: '&8&m----------&r &eGlobal Statistics for {player} &8&m----------'
  stats_kills: '&7💀 Total Kills: &e{kills} &7(across all servers)'
  stats_deaths: '&7☠️ Total Deaths: &e{deaths} &7(across all servers)'
  stats_kdr: '&7⚔️ Kill/Death Ratio: &e{kdr}'
  top_header: '&8&m----------&r &eGLOBAL TOP {type} LEADERBOARD &8&m----------'
  top_entry: '&7{rank}. &e{player}&7: &a{value}'

Troubleshooting

Common Issues

"Player not found" Error

  • Cause: Player hasn't played on current server but has stats on others
  • Solution: Fixed in v1.0.0 - now searches across all servers automatically

Database Connection Failed

  • Check: Database host, port, credentials, and network connectivity
  • Solution: Verify MariaDB is running and accessible from server

Statistics Not Syncing

  • Check: All servers using same database configuration
  • Solution: Ensure identical config.yml database settings on all servers

Debug Mode

Enable debug logging in config.yml:

debug: true
